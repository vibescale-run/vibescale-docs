<Warning>
  The Security Scan is only available after completing the **Opportunity Scale Review**. Please finish that step before proceeding.
</Warning>

## How to start

You can start the scan from two places:

- On the **Security Check** page, click **Start Scan**.
- On the main dashboard, click **Start Security Check**.

Both options trigger the exact same process.

## What happens during the scan

The scan takes about **20 seconds** and runs through four sequential steps. You can follow the progress in real time:

<Steps>
  <Step title="Code Analysis">
    Examines the repository's source code for insecure patterns, vulnerable logic, and exposure of sensitive data.

    Status shown: **Analyzing...**  **Completed**
  </Step>
  <Step title="Dependency Review">
    Checks all libraries imported by the project—including indirect dependencies—against known vulnerability databases.

    Status shown: **Analyzing...**  **Completed**
  </Step>
  <Step title="Secrets Detection">
    Scans the repository’s history and files for accidentally committed credentials, tokens, and API keys.

    Status shown: **Analyzing...**  **Completed**
  </Step>
  <Step title="Runtime Execution">
    Assesses the attack surface at runtime: server configurations, HTTP headers, and other risks that manifest in production.

    Status shown: **Analyzing...**  **Completed**
  </Step>
</Steps>

A progress bar and overall percentage are displayed throughout the scan. Completed steps show as **Completed**; upcoming steps show as **Waiting...**.

## After completion

When the scan finishes, you will see the message **"Security Check Completed!"** and the **View Security Report** button will be enabled.

<Tip>
  To rerun the scan after fixing vulnerabilities, use the **Re-Check** option—available in the report once all critical and high findings are resolved or dismissed.
</Tip>
