<Warning>
  The Security Scan is only available after completing the **Opportunity Scale Review**. Please complete that step first before proceeding.
</Warning>

## How to Start

You can start the scan from two places:

- On the **Security Check** page, click **Start Analysis**.
- On the main dashboard, click **Start Security Check**.

Both options trigger the exact same process.

## What Happens During the Analysis

The scan takes about **20 seconds** and runs through four sequential stages. You can monitor progress in real time:

<Steps>
  <Step title="Code Analysis">
    Examines the repository’s source code for unsafe patterns, vulnerable logic, and exposure of sensitive data.

    Status shown: **Analyzing...** → **Completed**
  </Step>
  <Step title="Dependency Review">
    Checks all libraries imported by the project — including indirect dependencies — against known vulnerability databases.

    Status shown: **Analyzing...** → **Completed**
  </Step>
  <Step title="Secrets Detection">
    Scans the repository’s history and files for accidentally committed credentials, tokens, and API keys.

    Status shown: **Analyzing...** → **Completed**
  </Step>
  <Step title="Runtime Execution">
    Assesses the attack surface at runtime: server configurations, HTTP headers, and other risks that manifest in production.

    Status shown: **Analyzing...** → **Completed**
  </Step>
</Steps>

A progress bar and overall percentage are displayed throughout the analysis. Completed stages show as **Completed**; upcoming stages show as **Waiting...**.

## After Completion

When the analysis finishes, you’ll see the message **"Security Check Completed!"** and the **View Security Report** button will be enabled.

<Tip>
  To rerun the analysis after fixing vulnerabilities, use the **Re-Check** — available in the report once all critical and high findings are resolved or dismissed.
</Tip>
