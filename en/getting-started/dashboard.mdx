The dashboard is the control center for your app. From here you start analyses, track progress in real time, and access the results of each step.

## App information

At the top of the dashboard you'll find the app name and description exactly as they were registered. These fields provide context for all generated reports.

## Analysis tracks

VibeScale organizes analysis into two independent tracks, each with its own steps and status indicators.

<CardGroup cols={2}>
  <Card title="Opportunity" icon="chart-line">
    **Discovery → Alignment → Report**

    Evaluates market maturity, product-market fit, and generates a report with positioning recommendations.
  </Card>
  <Card title="Security" icon="shield-check">
    **Code → Deps → Secrets → Runtime**

    Scans the source code, dependencies, exposed secrets, and the production attack surface.
  </Card>
</CardGroup>

## Status indicators

Each step in a track displays one of the following states:

| Status | Meaning |
|--------|---------|
| Pending | The step has not started yet. |
| In progress | The analysis is running. |
| Completed | Results are available. |

## Action buttons

- **Start opportunity analysis** — triggers the full Opportunity track (Discovery → Alignment → Report).
- **Start Security Check** — triggers the full Security track (Code → Deps → Secrets → Runtime).
- **Run Re-Check** — runs a new security check after fixes have been applied.

## Gate system

<Note>
  The **Security** track is only unlocked after the Opportunity analysis is complete. Similarly, the **Re-Check** is only available after all critical and high vulnerabilities identified in the previous check have been resolved. This sequencing ensures you act on the right information in the right order.
</Note>
